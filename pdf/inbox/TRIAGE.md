# PDF inbox triage

This file records first-pass decisions for newly downloaded PDFs.

The goal is **not** to promote every paper into a new sample.
The goal is to estimate where each paper is currently most useful:
- as inspiration for a canonical template
- as inspiration for a narrow variant
- as a distilled pattern note
- or simply as archive-only retention

These are soft placements, not irreversible decisions.

For the full simulation-heavy review across both archived and inbox papers, also see:
- `../SIMULATION_CURATION.md`

## Why most inbox papers still are not new samples

The inbox remains heavy on:
- review papers
- diagnostic composites
- apparatus-heavy experimental papers
- simulation papers whose main value is field+profile or workflow judgment rather than one clean reusable plotting grammar

These are often valuable scientifically, but promoting all of them into `samples/` would make retrieval worse.
The better strategy is to preserve the plotting judgment in `patterns/`, keep only a few strong scriptable starters, and use `../SIMULATION_CURATION.md` to remember why the remaining simulation papers still matter.
