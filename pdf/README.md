## Inbox

Newly downloaded PDFs should go to:

```text
pdf/inbox/
```

The repository root should stay free of loose PDFs.
If a PDF arrives at the root, move it into `pdf/inbox/` before doing anything else.

Use the inbox when papers are still untriaged.
Not every new PDF should become a normalized archive entry or a new plotting sample.

A new paper will often be most useful in one of four homes:
- a canonical sample
- a secondary variant sample
- a note in `patterns/`
- or archive-only retention with no promotion

See:
- `pdf/inbox/README.md` for the current inbox files
- `pdf/inbox/TRIAGE.md` for first-pass soft placement notes
- `pdf/inbox/NUMERICAL_PRIORITY.md` for the simulation-weighted shortlist
